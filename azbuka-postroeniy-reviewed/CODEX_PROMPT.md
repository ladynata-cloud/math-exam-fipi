# Промпт для Codex: интеграция «Азбуки построений» на сайт

Нужно выложить статический модуль `azbuka-postroeniy-reviewed` на сайт.

## Что сделать

1. Скопировать всю папку `azbuka-postroeniy-reviewed/` в публичную директорию сайта.
2. Проверить, что страница открывается по адресу `/azbuka-postroeniy-reviewed/index.html`.
3. Проверить, что iframe‑тренажёры из папки `trainers/` открываются с того же домена и не блокируются CSP.
4. Добавить ссылку на модуль в навигацию сайта под названием «Азбука построений».
5. Не менять математическую логику тренажёров без отдельного согласования.

## Важные файлы

- `index.html` — главная страница линейки.
- `manifest.json` — порядок тренажёров.
- `trainers/*.html` — сами тренажёры.
- `REVIEW_NOTES.md` — список уже внесённых UX‑правок.

## Ожидаемое поведение

- Ученик видит маршрут из 6 построений.
- Можно запускать отдельный тренажёр, отмечать попытки, назначать задания и экспортировать отчёт.
- Прогресс хранится в `localStorage` по ключу `azbuka-postroeniy-progress-v2`.
- На мобильном экране нижняя панель подсказок тренажёра остаётся видимой.

## Smoke test после выкладки

Открыть главную страницу и вручную проверить:

1. Первый тренажёр загружается внутри iframe.
2. Кнопка «Открыть отдельно» открывает текущий тренажёр.
3. «Засчитать попытку» увеличивает счётчик.
4. «Следующее незавершённое» переводит к следующей карточке.
5. Ссылка‑назначение формируется и импортируется обратно.

Не удалять папку `trainers/`, иначе iframe на главной странице перестанет работать.
