# Регламент работы ChatGPT + Claude + GitHub для Mathexam.space

Дата фиксации: 25 мая 2026

## Главная идея

Mathexam.space развивается не как хаотическая коллекция тренажёров, а как рабочая учебная среда, которая помогает конкретным ученикам и экономит время преподавателя.

Основной принцип:

```text
проблема ученика → методический маршрут → тренажёр → ссылка → тестирование → улучшение
```

## Роли инструментов

### ChatGPT

Основной штаб проекта.

Используется для:

- разработки стратегии;
- методической структуры;
- объяснений для учеников;
- написания и правки кода;
- работы с GitHub;
- публикации файлов на сайт;
- исправления багов;
- сборки маршрутов для учеников;
- доведения черновиков Claude до рабочей версии.

Формат работы:

```text
пользователь описывает задачу обычным языком → ChatGPT делает код/правку/публикацию → пользователь тестирует
```

### Claude.ai

Второй мастер: дизайнерско-методический прототипировщик.

Используется для:

- красивых первых HTML-прототипов;
- сложного педагогического UX;
- визуальных тренажёров в стиле GeoGebra / учебной доски;
- методических карт;
- альтернативного взгляда на задачу;
- 1–2 итераций по свежему короткому чату.

Не используется для:

- мелких CSS-правок;
- переименований;
- исправления одной кнопки;
- добавления ссылок на сайт;
- долгой правки одного большого HTML в старом чате.

Важное правило:

```text
один большой прототип → один свежий чат Claude → затем доводка через ChatGPT и GitHub
```

### Claude API через GitHub Actions

Технический запасной и черновой канал.

Уже настроена цепочка:

```text
GitHub Secret → GitHub Actions → Claude API → trainers/drafts/
```

Используется для:

- коротких тестов;
- черновиков в `trainers/drafts/`;
- будущей серийной генерации мини-тренажёров по готовому ТЗ;
- экспериментов с автоматизацией.

Пока не является основным каналом работы.

API credits расходуются отдельно от Claude Max. Auto-reload credits не включать. Пополнять вручную небольшими суммами.

### GitHub

Рабочая мастерская сайта.

Используется для:

- хранения HTML-тренажёров;
- публикации на GitHub Pages;
- черновиков Claude в `trainers/drafts/`;
- рабочих файлов в `trainers/`;
- документации проекта в `docs/`.

## Финансовая и лимитная стратегия Claude

Текущий оплаченный Claude Max 20× используется до окончания текущего периода как мощная ручная мастерская для красивых прототипов.

Следующий период понижен до Max 5×, потому что Claude планируется использовать как второго мастера, а не как главный производственный штаб.

Главный вывод:

```text
Max 20× нужен только при очень интенсивной ежедневной ручной работе в Claude.ai.
Max 5× вероятнее всего достаточно, если ChatGPT остаётся штабом, а Claude — прототипировщиком.
API credits нужны отдельно и используются осторожно.
```

## Как экономить лимит Claude

1. Один тренажёр — один новый чат.
2. Не править один большой HTML много раз в старом чате.
3. Сначала формулировать точное ТЗ.
4. Если нужен красивый прототип — просить полный HTML сразу.
5. После получения первого прототипа переносить доводку в ChatGPT/GitHub.
6. Не загружать большие PDF в длинные рабочие чаты без необходимости.
7. Не использовать Claude для рутинной технической возни.

## Рабочий цикл одного тренажёра

```text
1. Пользователь описывает проблему ученика.
2. ChatGPT предлагает минимальный учебный маршрут.
3. ChatGPT делает тренажёр сам или решает привлечь Claude.
4. Если нужен Claude — он создаёт первый прототип.
5. Черновик попадает в trainers/drafts/ или передаётся в чат.
6. ChatGPT проверяет, чистит и публикует в trainers/.
7. Пользователь тестирует на ученике.
8. По 2–5 замечаниям ChatGPT исправляет файл.
9. Готовый тренажёр добавляется на карту сайта.
```

## Приоритеты проекта на ближайший период

### 1. Информатика

Главный срочный фронт.

Цель: помочь ученику не сорваться на экзамене и освоить устойчивые шаблоны кода.

Приоритетные темы:

```text
1. Рекурсия: задачи 16 и 23.
2. Шаблон кода для подсчёта путей.
3. Файловая задача 17.
4. Строковая задача 24 или переборная задача 25.
5. Моделирование процесса в задаче 26.
6. Задача 27 — только если останутся силы.
```

### 2. Mathexam как учебный пульт

Цель: чтобы сайт работал как система быстрых ссылок для учеников.

Нужны маршруты:

- информатика;
- Маруся;
- неравенства;
- отрицательные числа;
- простые уравнения;
- самостоятельная работа, если урок сорвался.

### 3. Фабрика тренажёров

Цель: постепенно убрать ручную копипасту между ИИ-системами.

Текущий статус:

- Claude API подключён через GitHub Actions;
- секрет хранится в GitHub Secrets;
- тестовый запуск Claude через GitHub Actions прошёл успешно;
- результаты сохраняются в `trainers/drafts/`.

## Главное организационное правило

Каждый день — максимум один новый технический фронт.

Не распыляться между учениками, сайтом, Claude API, дизайном, Notion, GitHub и новыми идеями одновременно.

В первую очередь — живые ученики и ближайшие уроки.

Проект должен помогать не бросить учеников, а не съедать силы преподавателя.
